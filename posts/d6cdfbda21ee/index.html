aoaosnmp-SNMPv1 SNMPv2高危漏洞

发表于2023-03-27|更新于2023-03-27|AoaoIT技术安全
|阅读量:

使用到 SNMPv1 SNMPv2就是最高危漏洞，wiki下

漏洞情况

Vulnerability Result

public allows SNMPv2 access which is an obsolete version. public allows SNMPv1 access which is an obsolete version.

Vulnerability Description

Simple Network Management Protocol (SNMP) is an “Internet-standard protocol for managing devices on IP networks.”
The authentication of clients of earlier versions of SNMP is performed only by a “community string”, in effect a type of password, which is transmitted in cleartext.
The Internet Engineering Task Force (IETF) has designated SNMPv3 a full Internet standard, the highest maturity level for an RFC. It considers earlier versions to be obsolete designating them (“Historic”).

Detection Logic

QID Detection Logic (unauthentication):
This QID checks if the target allows SNMP access, if it is an end-of-life SNMP version the QID is posted.

Impact

The system is at high risk of being exposed to security vulnerabilities. Since the vendor no longer provides updates, obsolete software is more vulnerable to viruses and other attacks.

Solution

Disable or remove SNMPv1/2c authentication. Use SNMP version 3 authentication

SNMPv3 provides additional security features:
Confidentiality - Encryption of packets to prevent snooping by an unauthorized source.
Integrity - Message integrity to ensure that a packet has not been tampered with in transit including an optional packet replay protection mechanism.
Authentication - to verify that the message is from a valid source.

For IBM AIX users the following document is recommended SNMP for network management

If SNMP can not be used, look into third party applications / remote services that can be a replacement for SNMP.

Workaround:

As a temporary measure, block access to SNMP services at the network perimeter.

In situations where blocking or disabling SNMP is not possible,restrict all SNMP access to separate, isolated management networks that are not publicly accessible.

处理步骤

net-snmp-create-v3-user -ro -A password -x AES useraoao

snmp v1 v2 由于不安全，被PCI认证禁止，只能启用SNMP V3，现在把snmp的V3配置记录下来

snmp v1 v2c 关闭，要snmpd.conf把下面两行注释掉。

group notConfigGroup v1 notConfigUser

group notConfigGroup v2c notConfigUser

安装软件

yum install net-snmp net-snmp-devel net-snmp-utils

加只读用户snmpuser 密码为123456

net-snmp-config –create-snmpv3-user -ro -A 123456 -a MD5 snmpuser

用snmpuser用户来检查

snmpwalk -v 3 -u snmpuser -a md5 -A 123456 -l authNoPriv localhost .1 sysDescr

net-snmp-create-v3-user -ro -A H2kE3lAavyCtArMyx9yW -a SHA -X NBO8Tl4griZ9BViwvdi0 -x AES orionuser

snmpwalk -v 3 -u orionuser -a SHA -A H2kE3lAavyCtArMyx9yW -l authNoPriv localhost .1 sysDescr

snmpwalk -v 3 -u orionuser -l authPriv -a SHA -A H2kE3lAavyCtArMyx9yW -x DES -X NBO8Tl4griZ9BViwvdi0 localhost

snmpwalk -v 3 -u orionuser -l authPriv -a SHA -A NBO8Tl4griZ9BViwvdi0 -x DES -X H2kE3lAavyCtArMyx9yW localhost

snmp v3安全级别有三种，分别为noAuthNoPriv（不认证也不加密）、authNoPriv（认证但是不加密）、authPriv（既认证又加密）

systemctl status snmpd.service # 查看snmp服务当前状态

systemctl start snmpd.service # 启动snmp服务

systemctl enable snmpd.service # 设置开机启动snmp服务

firewall-cmd –list-all

firewall-cmd –permanent –zone=public –add-port=161/udp

systemctl restart firewalld.service

snmpwalk -v 2c -c public localhost .1.3.6.1.2.1.1
